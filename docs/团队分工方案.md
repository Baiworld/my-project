## 2. 成员详细分工

### 2.1 成员 A — 项目负责人 + 数据工程师（Data Lead）

| 类别 | 内容 |
|---|---|

| **技术栈** | Scala 2.13.10 / Spark 3.3.1 / Kafka Client / Spark MLlib / SparkSQL / JDBC |
| **精力分配** | 数据开发 60% + 架构管理 25% + 联调测试 15% |

**负责模块（Owner）：**

| 模块 | 具体交付 | 对应需求 | 状态 |
|------|---------|---------|:--:|
| `data-generator/` | 3 个 Topic 的静态模拟数据生成器 + 管道脚本（generate_data.py / feed_data.sh / start_all_v2.sh / Flume 配置），数据 ID 格式修正 | — | ✅ |
| `spark-streaming/` | RealTimeAnalysisApp + DataValidator + UserProfileAgg + ContentHotStats + ColdStartCluster + MySQLBatchWriter（含 content_metadata 写入） | FR-02~05 | ✅ |
| `spark-offline/` | OfflineAnalysisApp + UserPortraitBuilder + ALSTrainer（真实数据驱动）+ ALSRecommender + HybridRecommender + MetricsCalculator | FR-06~09 | ✅ |
| 推荐算法 | ALS 协同过滤（真实行为数据驱动的隐式反馈） + K-Means 冷启动聚类 + ε-greedy 探索（ε=0.15） + DPP 多样性重排（θ=0.7） + 过渡期线性插值 | FR-08 | ✅ |

**架构管理职责：**

- P1 结束前输出 **接口规范**（Kafka JSON Schema、MySQL 字段约定、WebSocket 数据格式、大屏指标定义）
- 所有 PR 必须经成员 A Review 通过方可合并
- 每日站会主持、每周周报、里程碑验收

**与各成员接口：**


## 8. 实际项目与原始计划的差异说明

以下为项目实际实现与原始分工方案的差异（在答辩时如被问到，可如实说明）：

| 差异项 | 原计划 | 实际实现 | 原因 |
|--------|--------|---------|------|
| 数据库表数 | 12 张 | 10 张 | 实时推荐表未创建、内容相似度表已移除 |
| 离线模块数 | 4 个 | 5 个（ALSTrainer + ALSRecommender 独立拆分） | 职责分离：训练和推理是两个独立作业 |
| Docker 环境 | 全员依赖 Docker Compose | 本地安装（CLAUDE.md 注明"不依赖 Docker"） | 降低环境复杂度，Windows 上 Docker 配置成本高 |
| CI/CD 流水线 | GitHub Actions 自动构建 | 无（手动编译运行） | P6 阶段优先级低于联调测试 |
| 单元测试 | pytest + ScalaTest | 无 | 开发周期紧张，以端到端验证替代 |
| A/B 测试框架 | 冷启动策略效果对比 | 无 | P6 阶段未实现 |
| Prometheus 监控 | 生产环境监控 | 无 | 当前为演示/开发环境，非生产 |
| 大屏图表数 | 7 图表（含热力图） | 7 图表 + 1 日志（含中国地图热力图） | 热力图已实现，基于 DataV geoJSON + 内容特征多样性 |
| Swagger 文档 | 完整 OpenAPI 文档 | 无（前后端通过代码直接对接） | P4 阶段优先交付 API 功能 |
| MySQL 版本 | MySQL 8.x | MySQL 5.7 | VM 环境限制，DATETIME(3) 默认值语法做兼容处理 |
| Kafka Producer | Scala Maven 模块 | Python data-generator 替代 | 简化数据生成，不依赖 Maven 编译 |
| ALS 训练数据 | 随机合成交互 | 真实行为数据驱动 | 基于 user_profile 的播放量/点赞率/收藏率等实际指标 |
| CTR 趋势计算 | WebSocket 与 HTTP API 两套公式 | 统一使用 row 级公式 | 修复 WebSocket 覆盖导致数据消失的 Bug |
| 启动脚本 | `start_all.sh` | `start_all_v2.sh`（虚拟机端）+ `start_app.bat`（Windows端） | 虚拟机与 Windows 分离部署 |
| 离线任务调度 | Cron 定时触发 | 手动按需运行（spark-submit job 参数） | 开发/演示环境无 Cron 基础设施 |
| 大屏图表数 | 7 图表 | 9 图表组件 | 新增 StrategyCompareChart、RegionHeatmap（中国地图） |
