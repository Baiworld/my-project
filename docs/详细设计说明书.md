### 3.1 实时数据流（在线链路）

```
时间线 ──────────────────────────────────────────────────────────────────────►

[用户]    [上游CMS]       [Kafka]              [Spark Streaming]      [Flask→Vue]
 │          │               │                      │                     │
 │ 点击播放  │               │                      │                     │
 ├─────────►│               │                      │                     │
 │          │ 写 Topic       │                      │                     │
 │          ├──────────────►│                      │                     │
 │          │               │ consumer.poll(500ms) │                     │

 │          │               │                      │ → UserProfileAgg    │
 │          │               │                      │ → ContentHotStats   │
 │          │               │                      │ → MySQL batchInsert  │
 │          │               │                      ├────────────────────►│
 │          │               │                      │                     │ MySQL 实时表
 │          │               │                      │                     │ rt_user_profile
 │          │               │                      │                     │ rt_content_hot
 │          │               │                      │                     │   │
 │          │               │                      │                     │   │ WebSocket

 │          │               │                      │                     │   └──► EventLogPusher
 │          │               │                      │                     │        滚动日志更新
```

### 3.2 离线数据流（批处理链路）

```
Cron 触发 (每 6 小时 / 每日凌晨)
│
├─ T+0min ── SparkSQL Job 启动
│   │
│   ├── MySQLSparkReader 读取 rt_user_profile (全量)
│   ├── MySQLSparkReader 读取 rt_content_hot (近 24h)
│   ├── MySQLSparkReader 读取 rt_coldstart_cluster (全量)
│   │
│   ├── T+30min ── UserPortraitBuilder 完成
│   │   └── 写入 offline_user_portrait
│   │
│   ├── T+75min ── HybridRecommender 完成
│   │   └── 写入 offline_recommendations
│   │   └── 传递推荐日志 → MetricsCalculator
│   │
│   └── T+90min ── MetricsCalculator 完成
│       └── 写入 offline_metrics
│
└─ T+120min ── Job 结束，日志归档
```

## 4. 🗄️ 数据库详细设计

### 4.1 命名规范

| 前缀 | 含义 | 读写特性 | 数据生命周期 |
|------|------|---------|-------------|
| `rt_` | Realtime 实时表 | 高频写（Spark Streaming batch insert） | 保留 7 天，定时清理 |
| `offline_` | Offline 离线表 | 批量写（SparkSQL），频繁读（Flask API） | 保留 90 天 |
| `sys_` | System 系统表 | 少量读写 | 永久保留 |

### 4.2 实时表 DDL

#### 📊 `rt_user_profile` — 用户实时行为画像

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | BIGINT UNSIGNED | PK, AUTO_INCREMENT | 主键 |
| `user_id` | BIGINT UNSIGNED | NOT NULL, INDEX | 用户 ID |
| `window_start` | DATETIME(3) | NOT NULL | 聚合窗口起始 |
| `window_end` | DATETIME(3) | NOT NULL | 聚合窗口结束 |
| `play_count` | INT UNSIGNED | DEFAULT 0 | 播放次数 |
| `completion_rate` | DECIMAL(5,4) | DEFAULT 0.0000 | 完播率 |
| `like_rate` | DECIMAL(5,4) | DEFAULT 0.0000 | 点赞率 |
| `favorite_rate` | DECIMAL(5,4) | DEFAULT 0.0000 | 收藏率 |
| `skip_rate` | DECIMAL(5,4) | DEFAULT 0.0000 | 跳过率 |
| `share_count` | INT UNSIGNED | DEFAULT 0 | 分享次数 |
| `comment_count` | INT UNSIGNED | DEFAULT 0 | 评论次数 |
| `preference_distribution` | JSON | NULL | 偏好风格/标签分布 `{"pop":0.35,"rock":0.20,...}` |
| `active_hours` | JSON | NULL | 活跃时段分布 `{"0":2,"1":0,...,"23":5}` |
| `is_cold_start` | TINYINT(1) | DEFAULT 1 | 是否冷启动用户 |
| `behavior_count` | INT UNSIGNED | DEFAULT 0 | 累计行为数 |
| `content_type_ratio` | JSON | NULL | 内容类型偏好比 `{"music":0.6,"video":0.4}` |
| `extended_attrs` | JSON | NULL | 扩展字段（预留） |
| `created_at` | DATETIME(3) | DEFAULT CURRENT_TIMESTAMP(3) | 创建时间 |

```sql
CREATE TABLE rt_user_profile (
    id           BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
    user_id      BIGINT UNSIGNED NOT NULL,
    window_start DATETIME(3) NOT NULL,
    window_end   DATETIME(3) NOT NULL,
    play_count   INT UNSIGNED DEFAULT 0,
    completion_rate DECIMAL(5,4) DEFAULT 0.0000,
    like_rate    DECIMAL(5,4) DEFAULT 0.0000,
    favorite_rate DECIMAL(5,4) DEFAULT 0.0000,
    skip_rate    DECIMAL(5,4) DEFAULT 0.0000,
    share_count  INT UNSIGNED DEFAULT 0,
    comment_count INT UNSIGNED DEFAULT 0,
    preference_distribution JSON DEFAULT NULL,
    active_hours JSON DEFAULT NULL,
    is_cold_start TINYINT(1) DEFAULT 1,
    behavior_count INT UNSIGNED DEFAULT 0,
    content_type_ratio JSON DEFAULT NULL,
    extended_attrs JSON DEFAULT NULL,
    created_at   DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    PRIMARY KEY (id),
    INDEX idx_user_id (user_id),
    INDEX idx_window_start (window_start),
    INDEX idx_is_cold_start (is_cold_start)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 6.1 应用入口配置

```scala
// RealTimeAnalysisApp.scala
object RealTimeAnalysisApp {
  def main(args: Array[String]): Unit = {
    val conf = Map(
      "spark.app.name"                -> "HybridRecSys-Streaming",
      "spark.master"                  -> "local[*]",
      "spark.streaming.batchDuration" -> "10",
      "spark.sql.shuffle.partitions"  -> "8",
      "spark.streaming.backpressure.enabled" -> "true",
      "spark.streaming.kafka.maxRatePerPartition" -> "500",
      "spark.streaming.stopGracefullyOnShutdown" -> "true"
    )

    val spark = SparkSession.builder()
      .appName(conf("spark.app.name"))
      .master(conf("spark.master"))
      .config("spark.sql.shuffle.partitions", conf("spark.sql.shuffle.partitions"))
      .getOrCreate()

    val ssc = new StreamingContext(spark.sparkContext, Seconds(10))
    ssc.checkpoint("hdfs://namenode:9000/checkpoints/streaming")

    // Kafka 配置
    val kafkaParams = Map[String, Object](
      "bootstrap.servers"  -> "localhost:9092",
      "key.deserializer"   -> classOf[StringDeserializer],
      "value.deserializer" -> classOf[StringDeserializer],
      "group.id"           -> "hybrid_rec_streaming",
      "auto.offset.reset"  -> "latest",
      "enable.auto.commit" -> "false"
    )

    val topics = Array("user_behavior", "content_metadata", "user_register")

    // 创建 DStream
    val kafkaStream = KafkaUtils.createDirectStream[String, String](
      ssc,
      PreferConsistent,
      Subscribe[String, String](topics, kafkaParams)
    )

    // 分流
    val behaviorStream  = kafkaStream.filter(_.topic() == "user_behavior").map(_.value())
    val metadataStream  = kafkaStream.filter(_.topic() == "content_metadata").map(_.value())
    val registerStream  = kafkaStream.filter(_.topic() == "user_register").map(_.value())

    // ── 管线组装 ──
    val validator = new DataValidator()

    // 校验三条流
    val validatedBehavior = behaviorStream.transform(rdd => validator.validateBehaviorRDD(rdd))
    val validatedRegister = registerStream.transform(rdd => validator.validateRegisterRDD(rdd))

    // 用户画像聚合 (5 min 滑动窗口, 2.5 min 滑动步长)
    val profileAgg = new UserProfileAgg(
      windowDuration = Minutes(5),
      slideDuration  = Minutes(2).+(Seconds(30)),
      coldStartThreshold = 50
    )
    val userProfileRDD = profileAgg.aggregate(validatedBehavior)

    // 内容热度统计 (1 min 滚动窗口)
    val hotStats = new ContentHotStats(
      windowDuration = Minutes(1),
      weightPlay      = 1.0,
      weightCompletion = 1.5,
      weightInteraction = 2.0
    )
    val contentHotRDD = hotStats.computeHotness(validatedBehavior)

    // 冷启动聚类 (每批次)
    val cluster = new ColdStartCluster(k = 8, maxIterations = 20)
    val clusterResultRDD = cluster.cluster(validatedRegister)

    // 批量写入 MySQL
    val writer = new MySQLBatchWriter(
      batchSize = 500,
      jdbcUrl   = "jdbc:mysql://localhost:3306/hybrid_rec?useSSL=false&rewriteBatchedStatements=true"
    )

    userProfileRDD.foreachRDD(rdd => writer.writeUserProfile(rdd))
    contentHotRDD.foreachRDD(rdd => writer.writeContentHot(rdd))
    clusterResultRDD.foreachRDD(rdd => writer.writeClusterResult(rdd))

    // 启动
    ssc.start()
    ssc.awaitTermination()
  }
}
```

### 6.4 ContentHotStats — 内容热度统计算法

```scala
// 热度分数计算公式
def calcHotScore(playCount: Long, completionRate: Double, interactionRate: Double): Double = {
  // 时间衰减因子 (越新的数据权重越高)
  val timeDecay = 1.0

  weightPlay        * math.log1p(playCount) * timeDecay +
  weightCompletion   * completionRate * 100.0 +
  weightInteraction  * interactionRate * 100.0
}

// 各权重默认值:
// weightPlay        = 1.0  (播放量 — 基础热度)
// weightCompletion  = 1.5  (完播率 — 内容质量信号)
// weightInteraction = 2.0  (互动率 — 用户认可信号，权重最高)
```

```scala
object UserPortraitBuilder {
  def build(spark: SparkSession, batchTime: String): Unit = {

    // ── Step 1: 从 MySQL 加载数据 ──
    val profileDF = spark.read
      .format("jdbc")
      .option("url", mysqlJdbcUrl)
      .option("dbtable", "rt_user_profile")
      .option("user", "reader")
      .option("password", env("MYSQL_PASSWORD"))
      .load()

    // ── Step 2: 读取内容元数据 (用于 TF-IDF 标签计算) ──
    val metaDF = spark.read
      .format("jdbc")
      .option("dbtable", "content_metadata")
      .load()

    // ── Step 3: TF-IDF 长期兴趣标签 ──
    // 从 preference_distribution JSON 展开 + 按 user_id 聚合所有窗口
    val tagDF = profileDF
      .select($"user_id", from_json($"preference_distribution", MapType).as("tags"))
      .groupBy("user_id")
      .agg(collect_list("tags").as("tag_history"))

    // TF-IDF: 用户对标签的"文档频率"(时间窗口) × 逆用户频率
    // 结果: Map[String, Double] → JSON 写入 long_term_tags

    // ── Step 4: 128 维偏好向量 ──
    // 综合行为特征 (播放/点赞/收藏/完播) × 内容类型 (音乐/视频) × 时间衰减
    // 使用 SVD/PCA 降维到 128 维, 或直接用多层特征拼接

    // ── Step 5: 生命周期阶段判定 ──
    val lifecycleDF = profileDF.groupBy("user_id")
      .agg(
        max("behavior_count").as("total"),
        max("created_at").as("last_active"),
        count(when($"created_at" > date_sub(current_date(), 30), lit(1))).as("active_days")
      )
      .select($"user_id",
        when($"total" <= 50, lit("new"))
          .when($"active_days" >= 20, lit("active"))
          .when($"active_days" >= 5, lit("silent"))
          .otherwise(lit("churned"))
          .as("lifecycle_stage")
      )

    // ── Step 6: 写入离线表 ──
    portraitDF.write
      .mode(SaveMode.Overwrite)
      .jdbc(mysqlJdbcUrl, "offline_user_portrait", mysqlProps)
  }
}
```

### 8.5 🛡️ 权限装饰器设计

```python
# permissions/decorators.py
from functools import wraps
from flask_jwt_extended import get_jwt, verify_jwt_in_request

def require_role(*roles: str):
    """
    RBAC 角色校验装饰器
    用法: @require_role('admin', 'operator')
    """
    def wrapper(fn):
        @wraps(fn)
        def decorator(*args, **kwargs):
            verify_jwt_in_request()
            claims = get_jwt()
            user_roles = set(claims.get("roles", []))
            required = set(roles)

            if not user_roles.intersection(required):
                return error("权限不足", 403, detail={
                    "required": list(required),
                    "current": list(user_roles)
                })

            return fn(*args, **kwargs)
        return decorator
    return wrapper
```

### 9.5 WebSocket Composable

使用 Socket.IO 客户端，支持自动重连。

```js
// composables/useWebSocket.js
import { ref, onUnmounted } from 'vue'
import { io } from 'socket.io-client'

export function useWebSocket() {
  const connected = ref(false)
  let socket = null

  function connect() {
    socket = io('/', { transports: ['websocket', 'polling'] })
    socket.on('connect', () => { connected.value = true })
    socket.on('disconnect', () => { connected.value = false })
  }

  function on(event, handler) { socket?.on(event, handler) }
  function off(event, handler) { socket?.off(event, handler) }
  onUnmounted(() => { socket?.disconnect() })

  return { connected, connect, on, off }
}
```

## 10. 📊 推荐算法详细设计

### 10.1 ALS 协同过滤 (存量用户)

```
输入: 用户-内容交互矩阵 R (隐反馈)
  - 播放 = 1.0
  - 完播 = 2.0
  - 点赞 = 3.0
  - 收藏 = 4.0
  - 分享 = 5.0
  - 跳过 = -1.0 (负反馈)
  - 无交互 = 0

参数:
  rank          = 50      // 隐因子维度
  regParam      = 0.1     // L2 正则化系数
  alpha         = 40.0    // 隐反馈置信度权重
  maxIter       = 10      // 最大迭代次数

输出:
  userFactors:  U × 50 矩阵
  itemFactors:  I × 50 矩阵

推荐分数:
  score(u,i) = dot(userFactors[u], itemFactors[i])
```

### 10.2 Item2Vec 内容向量召回

```
训练样本构建:
  - 取每个用户的交互序列 (按时间排序)
  - 滑动窗口 size=5, 生成 (content_i, content_j) 共现对
  - 音乐和视频在同一个向量空间中训练 (共享 embedding)

Word2Vec 参数:
  vectorSize = 64
  windowSize = 5
  minCount   = 3
  negative   = 5
  iterations = 10

召回:
  - 取用户最近交互的 20 条内容
  - 查询每条的 embedding
  - 取平均作为用户实时兴趣向量
  - 在内容 embedding 空间做 ANN 检索 (Top-100)
```

### 10.3 DPP 多样性重排序

```
DPP (Determinantal Point Process) 核心思想:
  在保证相关性的前提下, 最大化推荐集合的"体积"(多样性)

物品核矩阵 L:
  L[i][j] = q(i) × sim(i,j) × q(j)

  其中:
    q(i) = exp(score(i)) / Σ exp(score)  // 归一化相关性分数
    sim(i,j) = 内容多维特征相似度 (标签 Jaccard + 风格/语言/BPM/时长)

求解 (贪心近似):
  selected = []
  while len(selected) < N:
    i* = argmax_{i∉selected} log det(L[selected ∪ {i}])
    selected.append(i*)

  实际实现中, 取 Top-100 候选做 DPP 重排:
    - 音乐候选 60 条 + 视频候选 40 条 = 100 条
    - DPP 重排后取 Top-50
```

### 10.4 跨模态加权融合公式

```
最终推荐分数 (存量用户):

  final_score(u, i) =
      0.50 × ALS_score(u,i)                           // 协同过滤
    + 0.30 × Item2Vec_score(u,i)                      // 内容召回
    + 0.10 × hot_score(i)                             // 热度
    + 0.05 × (1 - avg_similarity(i, selected))        // 多样性奖励
    + 0.05 × content_type_match(u, i)                 // 类型偏好匹配

  其中 content_type_match:
    = user_content_type_ratio[content_type(i)]        // 用户的音乐/视频偏好比

冷启动推荐分数:
  final_score(u, i) =
      0.35 × cluster_hot_score(i)                     // 同簇热门
    + 0.30 × tag_match_score(u.interest_tags, i.tags) // 标签匹配
    + 0.20 × global_hot_score(i)                      // 全局热门
    + 0.15 × exploration_bonus(i)                     // ε-greedy 探索

过渡期权重平滑:
  ratio = (behavior_count - 30) / 50                  // ratio ∈ [0, 1]
  final_score = (1-ratio) × coldstart_score + ratio × existing_score
```


### 11.2 API 请求安全链路

```
每一次 API 请求:

  1. Vue Axios 拦截器:
     - 从 AuthStore 读取 access_token
     - 注入 Authorization: Bearer <access_token>

  2. 请求到达 Flask:
     - @jwt_required() 装饰器验证 JWT 签名 (HS256)
     - 检查 exp (过期), iat (签发时间)
     - 解析 sub, roles 到请求上下文

  3. @require_role(*roles) 装饰器:
     - 从 JWT claims 取出 roles
     - 与路由所需 role 做交集
     - 交集为空 → 403 Forbidden
     - 交集非空 → 放行

  4. CORS 校验:
     - Origin 头必须在 ALLOWED_ORIGINS 白名单内
     - 预检请求 OPTIONS 返回允许的 Method/Headers

  5. 参数化查询:
     - 所有 SQL 查询使用 SQLAlchemy ORM / 参数绑定
     - 禁止字符串拼接 SQL
```

### 11.3 密码策略

| 策略 | 要求 |
|------|------|
| 最小长度 | 8 字符 |
| 字符复杂度 | 大写字母 + 小写字母 + 数字 + 特殊字符，至少满足 3 类 |
| 哈希算法 | bcrypt (cost factor = 12) |
| 登录失败限制 | 连续 5 次失败锁定 15 分钟 |
| 密码历史 | 不允许与最近 3 次密码相同 |
